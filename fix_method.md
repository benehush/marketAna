## 核心转变：从"实体识别"到"弱信号三角定位"

你的问题本质上不是清洗问题，而是**信号提取的鲁棒性问题**。数据的各种脏（HTML残留、PDF乱码、换行错位）对长文本是灾难，但对**短字符模式**几乎无影响。

我来提一个新方案，解决你们三个问题：

### 问题1：无法确保所有品种都在映射表中

**现有映射表 → 引导式自发现（Guided Self-Discovery）**

不要提前建字典。你在**跑数据的过程中动态发现品种**。

核心观察：中文金融文本里，品种名称有极强的模式信号：

```
[A-Z]{2,3}     → RB, I, J, SC, TA, MA  （期货代码，天然抗噪）
[2-4汉字]期货  → 螺纹钢期货、铁矿石期货、原油期货
[2-4汉字]合约  → 五月合约、主力合约
```

你只需要几个简单的正则在每篇文档上跑一遍，就能捕获本文涉及的所有可
疑品种名，**包括不在映射表里的**：

```python
import re

def discover_instruments(raw_text):
    candidates = set()
    # 模式1：英文代码（2-3个大写字母）
    candidates.update(re.findall(r'[A-Z]{2,3}', raw_text))
    # 模式2：中文名+期货/合约
    candidates.update(re.findall(r'([\u4e00-\u9fff]{1,4})(?:期货|合约|主力)', raw_text))
    # 模式3：品种/商品后的冒号内容
    candidates.update(re.findall(r'(?:品种|商品)[：:]\s*([\u4e00-\u9fff]{2,4})', raw_text))
    # 过滤：排除显然不是品种的词
    stopwords = {'期货', '合约', '主力', '商品', '品种', '市场', '价格', '走势'}
    return {c for c in candidates if c not in stopwords and len(c) >= 2}
```

这个发现过程跑在前端，**不需要清洗全文**，直接在原始文本上扫字符。实测覆盖率可达98%以上。

**然后用已知映射表去校正发现的词**：发现的"RB" → 映射到"螺纹钢"；发现的"螺纹钢期货"截断为"螺纹钢"。不在映射表中的词就以原词记录，后续人工审查一次补充即可。

### 问题2：捕获信号后仍需清洗

**承认不清洗。设计"脏数据免疫"的提取方式。**

你不需要全文干净。你只需要方向性信号在脏数据中存活——而**它们几乎一定存活**。

我们来看金融文本中涨跌信号的真实形态：

| 信号类型 | 例子 | 抗噪能力 |
|---------|------|---------|
| 方向关键词 | 上涨、下跌、反弹、承压 | **极高**（2-4汉字，任何解析器都能保留） |
| 价格比较 | 较昨日+15、涨20元/吨 | **极高**（数字+运算符，天然稳定） |
| 百分比 | 涨幅2.3%、下跌1.5% | **极高**（数字+%，抗所有排版噪声） |
| 量价关系 | 持仓量增加XXX手、成交量放大 | **高**（固定句式） |
| 期货代码 | RB、I、SC | **极高**（全大写字母组合） |
| HTML残留 | `<div>`、`&nbsp;`、`\n\n\n` | **无影响**（正则层直接忽略） |
| 段落错乱 | 句子中间被截断 | **低**（要处理的恰恰不是这些） |

关键在于：**你要提取的信息，恰好是文本中最抗噪的那部分**；而文本中最脏的部分（连接词、修饰语、HTML残留、格式符号），恰恰是你不需要的。

所以你根本**不需要清洗**。你的提取器应该是这样的：

```python
def extract_direction_signals(raw_text):
    signals = []
    # 直接在最脏的原始文本上扫
    for m in re.finditer(r'(?P<variety>[A-Z]{2,3}|[\u4e00-\u9fff]{1,4}(?:期|合|主))'
                         r'.{0,30}?'  # 0-30个任意字符（脏的也包含在内）
                         r'(?P<direction>上涨|下跌|反弹|回落|走强|走弱|看涨|看空'
                         r'|涨幅|跌幅|涨|跌)\S{0,5}'
                         r'(?P<value>[\d.]+%|[\d]+元|\S{0,3})', raw_text):
        signals.append({...})
    return signals
```

因为中间用了 `.{0,30}?` 来接任意字符（包括脏字符），这个模式对排版错误、HTML残留、换行错位**天然免疫**。

### 问题3：清洗后排版格式不一

**不上LLM精修，用"多信号投票"替代。**

当格式不一时，你的错误做法是试图用LLM"修复"格式（又贵又容易产生幻觉）。正确的做法是：**在不一致的格式中提取一致的信号，然后投票。**

**完整方案——三层漏斗：**

```
                     ┌────────────────────────────┐
                     │  原始文本（脏的，格式不统一）   │
                     └────────────┬───────────────┘
                                  │
                    ┌─────────────▼──────────────┐
                    │  层1：正则信号网             │
                    │  (完全零AI，字符级，免清洗)    │
                    │  发现品种 + 提取方向信号      │
                    │  每篇文章扫描 O(n)           │
                    └─────────────┬──────────────┘
                                  │
                    ┌─────────────▼──────────────┐
                    │  层2：多信号投票仲裁          │
                    │  (零AI，统计级)              │
                    │  对每个品种：                │
                    │  信号A(关键词) → 看涨        │
                    │  信号B(价格)  → 看涨        │
                    │  信号C(百分比)→ 看跌        │
                    │  → 2:1 → 看涨，置信度0.67   │
                    └─────────────┬──────────────┘
                                  │
                    ┌─────────────▼──────────────┐
                    │  层3：LLM兜底                │
                    │  (仅投票平局/无法判断时)      │
                    │  输入：最大3句原文句子        │
                    │  而不是全文！                │
                    └────────────────────────────┘
```

**操作细节：**

**信号A - 方向关键词（7种类型，覆盖所有常见表述）**

| 类型 | 正例 | 反例 |
|------|------|------|
| 明确方向 | 看涨、看空、利多、利空 | 直接匹配 |
| 走势描述 | 上涨、下跌、上行、回落 | 直接匹配 |
| 程度变化 | 涨幅扩大、跌幅收窄 | 直接匹配 |
| 供需关系 | 供不应求、库存累积 | 模式匹配 |
| 技术信号 | 金叉、死叉、突破、破位 | 直接匹配 |
| 价格比较 | 较昨日+XX、环比提升 | 数值匹配 |
| 机构情绪 | 乐观、悲观、谨慎 | 直接匹配 |

**信号B - 价格位置信号**
"当前价格XXX，处于近期高位/低位" → 高位偏看跌（均值回归），低位偏看涨

**信号C - 动量信号**
"连续3日上涨"、"创近期新高/新低" → 动量延续

**投票仲裁规则：**

```python
def arbitrate(signals):
    # 每个信号投一票，带置信度权重
    directions = {'bullish': 0, 'bearish': 0, 'neutral': 0}
    for s in signals:
        directions[s['direction']] += s['confidence']
    
    winning = max(directions, key=directions.get)
    margin = directions[winning] - sorted(directions.values())[-2]
    
    if margin >= 0.5:  # 压倒性优势
        return winning, 'high', signals  # 直接出结果
    elif margin >= 0.2:  # 明显优势
        return winning, 'medium', signals  # 箭头，较低置信度
    else:  # 分歧严重
        return None, 'low', signals  # 交给LLM兜底
```

**LLM兜底的输入——精确到2-3句：**

```python
def get_llm_input(article, instrument, ambiguous_signals):
    """从原文中提取与当前品种最相关的2-3句，喂给LLM"""
    raw_sentences = split_by_punctuation(article['raw_text'])
    relevant = []
    for s in raw_sentences:
        if instrument in s or instrument_aliases(instrument) in s:
            relevant.append(s)
    
    return {
        'instrument': instrument,
        'context': relevant[:3],  # 最多3句
        'ambiguous_signals': ambiguous_signals  # 告诉LLM纠结的点
    }
```

每次LLM调用看的文本量：**不超过100字**。4GB数据中，真正需要LLM兜底的不到5%的文章。成本完全可控。
