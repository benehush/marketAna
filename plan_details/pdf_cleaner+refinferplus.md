# PDF Cleaner 与 Refiner 安全增强计划

## Summary
- 针对五矿这类长篇多品种 PDF，清洗策略保持“保留真实品种观点”，但新增 PDF 尾部免责声明截断和错位表格残句过滤。
- Refiner 从“积极编辑”改成“保守润色”：只顺句、轻量整理，不解释、不纠错、不补数字、不扩写。
- 精修输出增加安全校验；一旦模型输出包含自我解释、纠错注释或明显扩写，判定失败并保留既有 `refined_text`，不影响后续分析。

## Key Changes

### Cleaner
- 在 `pn05/structured_cleaner.py` 的结构化清洗流程中增加“免责声明块删除”：
  - 对 `body/ocr` 正文行生效。
  - 命中以下任一尾部信号后，删除该行及后续连续免责声明/公司信息块：
    - `期货有限公司是经中国证监会批准`
    - `本刊所有信息均建立在可靠`
    - `不构成买卖建议`
    - `不构成任何投资建议`
    - `未经书面许可`
    - `研究报告不代表协会观点`
    - `粤海街道`、`金融大厦` 等公司地址类尾部。
  - 不删除正文中正常出现的机构名称，只有进入免责声明语境后才截断。

- 增强 PDF 表格/分栏错位残句过滤：
  - 删除只有单位、数字、字段残片的行，例如：
    - `吨, 环比增加5144 吨...`
    - `当日注册仓...`
    - `环比变化至36.62 万手`
    - 以 `吨。`、`面,`、`变化至` 等残缺片段开头且缺少判断语义的行。
  - 保留包含完整分析判断的行，尤其含 `预计、建议、维持、关注、供应、需求、库存、成本、基差、利润、开工率、震荡、偏弱、偏强、风险` 等信号的行。
  - 保留多品种日报中的真实品种段落，不做全局摘要式压缩。

- 扩展 `CleanConfig` 可调项：
  - `drop_disclaimer_blocks: bool = True`
  - `drop_pdf_fragment_lines: bool = True`
  - `pdf_fragment_min_cjk_chars` 或等价阈值，用于控制残句过滤强度。
  - 默认开启，允许后续针对特殊报告回退。

### Refiner
- 收紧 `SYSTEM_PROMPT`：
  - 明确禁止输出 `注：`、`原文可能有误`、`建议核实`、`数据缺失`、`逻辑存疑`、`自我修正`、`更正：`。
  - 禁止纠错、补全缺失内容、推断数字、改写方向判断。
  - 禁止 Markdown 过度美化；保留原有标题即可，不新增项目符号、解释段或摘要。
  - 要求输出长度原则上不超过输入，除非只是必要的标点和语序修复。

- 增加精修结果校验：
  - 在 `_sanitize_refined_text()` 后增加 `_validate_refined_text(refined, source_chunk)`。
  - 若命中禁用短语，判定该 chunk 精修失败。
  - 若输出长度超过输入的合理阈值，例如 `len(refined) > len(source_chunk) * 1.15 + 200`，判定失败。
  - 若输出为空、只有包装语、或包含模型自述，判定失败。
  - 失败时写入 `TaskLog(stage="refiner", status="failed")`，不覆盖既有 `refined_text`。

- 精修失败策略保持 best-effort：
  - 任一 chunk 失败则整篇不写入新的 `refined_text`。
  - `cleaned_text` 始终不变，规则识别和证据定位继续使用 `cleaned_text`。

## Test Plan
- 新增 cleaner 单元测试：
  - PDF 尾部免责声明从“期货有限公司是经中国证监会批准”开始被删除。
  - `本刊所有信息`、`不构成买卖建议`、`未经书面许可`、`研究报告不代表协会观点`、公司地址均不出现在 `cleaned_text`。
  - 错位残句如 `吨, 环比增加5144 吨`、`环比变化至36.62 万手` 被删除。
  - 完整判断句仍保留，如 `预计成材价格将维持底部震荡走势`、`关注库存去化速度`、`供应偏宽松`。

- 新增 refiner 单元测试：
  - Fake LLM 返回正常润色文本时写入 `refined_text`。
  - Fake LLM 返回 `注：原文可能有误`、`自我修正`、`更正：` 时拒收并记录失败日志。
  - Fake LLM 输出明显长于输入时拒收。
  - 拒收时不覆盖已有 `refined_text`，文章状态仍保持 `CLEANED`。

- 回归测试：
  - `pn05/test_cleaner.py`
  - `pn11/test_pipeline.py`
  - `tests/test_backend_data.py`
  - 手动运行 `tests/manual_single_file_pipeline.py data/20250401/五矿期货_323248.PDF --output-dir tests/outputs`，确认：
    - `02_cleaned_text.txt` 不再包含免责声明和公司页脚。
    - 错位表格残句减少。
    - `04_refined_text.txt` 不再出现模型注释、自我纠错或数字推断。

## Assumptions
- 五矿这类多品种 PDF 的主体内容应保留，不追求压缩到单品种摘要长度。
- Cleaner 只删除噪声和明显残片，不在清洗阶段重写观点。
- Refiner 是用户展示层，不参与规则识别；安全优先于可读性，宁可精修失败回退，也不允许模型擅自改事实。
